#### 実務最適化
1. **graceful shutdown**: `SIGTERM`/`SIGINT`受信時にCLI子プロセスを確実に終了しゾンビ化を防止。3秒タイムアウト付き
2. **WebSocket自動再接続**: サーバー切断時に2秒→3秒→5秒→10秒の段階的間隔で最大10回自動再接続。意図的な切断（code 1000）時は再接続しない
3. **start.sh相対パス化**: `cd /Users/.../Code-Lab` → `cd "$(dirname "$0")"` でフォルダ移動しても動作
4. **仕様書ポート修正**: 9225 → 3456 に統一

#### 秘書チャット改善
1. **コピーボタン**: 秘書チャットタイトル右に「📋 コピー」ボタン。最新の秘書回答をクリップボードにコピー
2. **Cmd+A チャット内選択**: 質問モード時にCmd+Aでチャット内テキストのみを選択（画面全体ではなく）
3. **stream-json修正**: `--verbose`フラグ追加 + `result`型の重複テキスト除去で回答の重複を解消

#### その他

#### 追加機能
1. **モード選択UI**: コマンドバーに「📋 開発」「💬 質問」ボタンを設置。入力欄は初期非表示で、モード選択後に表示される。「←」ボタンでモード選択画面に戻る
2. **秘書エージェント**: Canvas上に社長の左隣（gx:6, gy:8.5）に秘書キャラクターを配置。女性キャラ（ダークブラウンのロングヘア、紫シャツ、スケール1.2）。専用のデスク・椅子・モニターあり
3. **秘書チャットパネル**: サイドパネルに秘書チャットセクションを追加。ユーザーの質問は右寄せ（薄アクセント背景）、秘書の回答は左寄せ（薄紫背景）で表示
4. **サイドパネル切り替え**: 開発モード → AI従業員一覧表示/秘書チャット非表示。質問モード → AI従業員一覧非表示/秘書チャット表示。Canvasのキャラクターは常時表示（消えない）
5. **双方向モード誤り検出**: 開発モードで質問っぽい入力をすると「質問のようです。質問モードに切り替えますか？」の確認バーが表示。質問モードで開発指示を入力すると逆の確認バーが表示。Regex（QUESTION_KEYWORD_RE / DEV_KEYWORD_RE）で検出
6. **デモ/Liveモード対応**: デモモードでは定型回答（「ご質問ありがとうございます。Liveモードに接続すると、詳しい回答が可能です💡」）。Liveモードでは`claude -p`に質問を送信し、実際の回答を返す

#### バグ修正
- 質問モードでファイル添付ボタン（＋）が消える問題を修正（`display:'none'` → `display:'flex'`）

## 技術メモ

### モード選択の仕組み
- `currentMode` 変数: `'dev'` / `'ask'` / `null`（初期状態）
- モード選択はボタンがトリガー（テキスト内容では判定しない）
- 送信時に `currentMode` を参照して処理を分岐

### 秘書のCanvas描画
- `drawCharacter()` の第6引数 `isSecretary` で女性キャラ描画を分岐
- スケール: 秘書1.2 / 社長1.3 / 通常1.15
- 髪色: `#4a2c2c`（ダークブラウン）、頭部円の下に矩形でロングヘア表現
- シャツ色: `#7c4dff`（紫）

### GitHub Pages デプロイ
- mainブランチのルートディレクトリからPages配信
- Live関連UIは自動で非表示（`location.hostname !== 'localhost'` 判定）
- デモモードのみ動作
