# Code-Lab — AI開発チーム指揮シミュレーター 完全仕様書 v3

## 概要

ブラウザ上で動作するアイソメトリック3D風の「AI開発チーム指揮アプリ」。
一人社長（ユーザー）がAI従業員7名+秘書エージェント1名に指示・質問を出し、
オフィスで作業する様子をリアルタイムにビジュアライズする。

**コンセプト**: 一人社長のAI開発事務所。社長が開発指示を出すとAI従業員が自律的に働き、質問をすると秘書が回答する。
**ビジュアル**: アイソメトリック（斜め見下ろし）視点。フラットデザイン+柔らかい影。
**動作モード**: デモモード（スタンドアロン）+ Liveモード（Claude Code CLI連動）

---

## 技術スタック

```
フロントエンド: 単一HTMLファイル（HTML + CSS + JavaScript、約2800行）
描画: Canvas API（アイソメトリック2.5D描画）
通信: WebSocket（Liveモード時）
フォント: Google Fonts（Inter, Noto Sans JP, JetBrains Mono）
バックエンド: Node.js 中継サーバー（server.js）
外部ライブラリ: なし（Pure JS）
```


## 3. Canvas描画仕様（アイソメトリック3D風）

### 3.1 描画基本設定

- **アイソメトリック座標系**: X軸=右下30°、Y軸=左下30°、Z軸=上方向
- 変換関数:
  ```javascript
  function toIso(x, y, z) {
    return {
      sx: (x - y) * TILE_W / 2 + OFFSET_X,
      sy: (x + y) * TILE_H / 2 - z * Z_SCALE + OFFSET_Y
    };
  }
  ```
- タイルサイズ: `TILE_W = 64, TILE_H = 32`（2:1比率のダイヤモンド型）
- オフィスグリッド: 12×10タイル
- 60fps描画（requestAnimationFrame）
- **深度ソート**: Y座標+Z座標でオブジェクトをソートして描画順を決定

### 3.2 ビジュアルスタイル

- **フラットデザイン**: シンプルな面、グラデーション少なめ
- **柔らかい影**: 各オブジェクトの下に楕円形のドロップシャドウ
- **パステル〜ナチュラル系の配色**: 温かみのあるオフィス感
- **線なし（ノーアウトライン）**: 面の色の差で形を表現
- **角丸**: `quadraticCurveTo`使用。**`roundRect`は使用禁止**

## 6. 質問処理フロー

### 6.1 デモモード

1. ユーザーが質問モードで質問を送信
2. 質問テキストを秘書チャットに右寄せ表示
3. 秘書 `working=true`, Canvas上アニメーション開始
4. 2〜3秒後に定型回答を左寄せ表示:
   - 「ご質問ありがとうございます。Liveモードに接続すると、詳しい回答が可能です💡」
5. 秘書 `working=false`

### 6.2 Liveモード

1. ユーザーが質問モードで質問を送信
2. `ws.send({ type:'question', text:'...' })` で送信
3. server.js の `handleQuestion(ws, msg)`:
   - `claude -p "以下の質問に簡潔に回答してください: [質問]"` を実行
   - `--output-format stream-json` で結果を受信
   - dev_eventは発行しない（7名のAI従業員は動かない）
   - `{ type:'question_answer', text:'...' }` で回答を返送
4. index.html: 秘書チャットに回答を左寄せ追加
5. 秘書 `working=false`

## 7. 開発処理フロー（インタラクティブデモ）

### 7.1 デモモード

1. ユーザーが開発モードで指示を送信
2. 全8エージェント（リード→デベロッパー→UIデザイナー→テスター→レビュアー→デバッガー→QA→リード）が連鎖で動作
3. 各エージェントが入力内容に応じた思考・作業をシミュレーション
4. 統計値（ファイル数、行数、テスト数、バグ数）が更新
5. 確認依頼が発生する場合あり

### 7.2 Liveモード

1. ユーザーが開発モードで指示を送信
2. WebSocket経由で server.js に送信
3. Claude Code CLI が実際にコード生成
4. ログ・イベントがリアルタイムにCanvasとサイドパネルに反映

## 9. Live接続アーキテクチャ

### 9.1 構成

```
[ブラウザ Code-Lab]  ←WebSocket→  [Node.js 中継サーバー(server.js)]  ←子プロセス→  [Claude Code CLI]
```

### 9.2 server.js 仕様

- WebSocketサーバー（ws モジュール）
- ポート: 3456
- メッセージタイプ:
  - `command` → Claude Code CLI に開発指示を送信
  - `question` → Claude Code CLI に質問を送信（`handleQuestion()`）
  - `abort` → 実行中のCLIプロセスを停止
  - `dev_event` → CLI出力をブラウザにリアルタイム転送
  - `question_answer` → 質問の回答をブラウザに返送
- **Graceful Shutdown**: `SIGTERM`/`SIGINT`受信時に全CLI子プロセスを終了しゾンビ化を防止。3秒タイムアウト付き強制終了

### 9.3 WebSocket自動再接続

- サーバー切断時（意図的切断=code 1000以外）に自動再接続を試行
- 再接続間隔: 2秒→3秒→5秒→5秒→10秒（段階的に拡大）
- 最大再接続回数: 10回。超過時は手動再接続を促すメッセージ表示
- 接続成功時にカウンタをリセット
- ユーザーが手動切断した場合（code 1000）は再接続しない

### 9.4 GitHub Pages対応

- `location.hostname !== 'localhost'` でLive関連UIを自動非表示
- デモモードのみ動作（WebSocket接続なし）

### 9.5 起動スクリプト（start.sh）

- Claude Code環境変数（`CLAUDECODE`等）を除去してからサーバーを起動
- 相対パス対応: `cd "$(dirname "$0")"` でフォルダ移動しても動作

## 13. 技術的注意事項（必読）

- **`c.roundRect()` は使用禁止** → `quadraticCurveTo` で角丸を描くこと
- 描画関数内では引数名を統一し、未定義変数の参照を絶対に起こさない
- Canvas描画ループ全体を try-catch で保護すること
- アイソメトリック描画の深度ソートを必ず実装
- 全UI・全メッセージ・全ログは日本語で表記すること
- GitHub Pages デプロイ時はLive関連UIを自動非表示にすること
