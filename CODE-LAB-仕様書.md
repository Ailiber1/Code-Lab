# Code-Lab — AI開発チーム指揮シミュレーター 仕様書 v2

## 概要

ブラウザ上で動作するアイソメトリック3D風の「AI開発チーム指揮アプリ」。
一人社長（ユーザー）がAI従業員7名に開発指示を出し、
オフィスで作業する様子をリアルタイムにビジュアライズする。

**コンセプト**: 一人社長のAI開発事務所。社長が指示を出し、AI従業員が自律的に働く。
**ビジュアル**: アイソメトリック（斜め見下ろし）視点。フラットデザイン+柔らかい影。
**最終目標**: Claude Code（ターミナル）と連動し、実際のコード生成をビジュアライズする。

本仕様書では「Phase 1: スタンドアロン版」を構築し、
「Phase 2: Claude Code連動」への接続口を設計段階から組み込む。

---

## 技術スタック

```
フロントエンド: 単一HTMLファイル（HTML + CSS + JavaScript）
描画: Canvas API（アイソメトリック2.5D描画）
通信: WebSocket（Phase 2用、Phase 1ではモック）
フォント: Google Fonts（Inter, Noto Sans JP, JetBrains Mono）
外部ライブラリ: なし（Pure JS）
```

### 2. Canvas描画仕様（アイソメトリック3D風）

#### 2.1 描画基本設定

- **アイソメトリック座標系**: X軸=右下30°、Y軸=左下30°、Z軸=上方向
- 変換関数:
  ```javascript
  function toIso(x, y, z) {
    return {
      sx: (x - y) * TILE_W / 2 + OFFSET_X,
      sy: (x + y) * TILE_H / 2 - z * Z_SCALE + OFFSET_Y
    };
  }
  ```
- タイルサイズ: `TILE_W = 64, TILE_H = 32`（2:1比率のダイヤモンド型）
- オフィスグリッド: 12×10タイル
- 60fps描画（requestAnimationFrame）
- **深度ソート**: Y座標+Z座標でオブジェクトをソートして描画順を決定

#### 2.2 ビジュアルスタイル

**ピクセルアートではない。** 以下のモダンスタイルを採用:

- **フラットデザイン**: シンプルな面、グラデーション少なめ
- **柔らかい影**: 各オブジェクトの下に楕円形のドロップシャドウ
- **パステル〜ナチュラル系の配色**: 温かみのあるオフィス感
- **線なし（ノーアウトライン）**: 面の色の差で形を表現
- **角丸**: 家具やUIは角丸を多用（`quadraticCurveTo`使用。`roundRect`は**使用禁止**）

### 3. キャラクター仕様

#### 3.1 共通仕様

- **描画**: アイソメトリック視点での後ろ姿（デスクに向かって座る）
- **構成**: シンプルな人型シルエット（頭+胴+腕+脚）
- **座位**: 椅子に座った状態で常時描画
- **各キャラに固有色**: 服の色で識別

#### 3.2 アニメーション

| 状態 | アニメーション |
|------|-------------|
| タイピング中 | 腕が微動、体が小さく上下（bob）、モニター画面にコード点滅 |
| 待機中 | 静止、モニター画面はスクリーンセーバー風 |
| 思考中 | 頭上に吹き出し表示（独り言テキスト） |
| Phase待機 | 椅子に座ったまま薄暗く表示（非アクティブ） |

#### 3.3 頭上表示

**役割タグ**: キャラ頭上に常時表示
- アクティブ時: 固有色で明るく（α=0.95）
- 非アクティブ時: グレーで薄く（α=0.4）
- 角丸バッジ形状

**思考バブル**: 作業中のみ表示
- 白い吹き出し（角丸矩形 + 吹き出しの尻尾）
- **`quadraticCurveTo`で描画。`roundRect`は使用禁止**
- テキスト: 作業に関する独り言（日本語）
- キャラ固有色でボーダー

**確認待ちアイコン**: 📩 が点滅（confirmQueueに未処理あり時）

### 7. 重要な技術的注意事項（必読）

- **`c.roundRect()` は使用禁止** → `quadraticCurveTo` で角丸を描くこと
- 描画関数内では引数名を統一し、**未定義変数の参照を絶対に起こさない**
  - 関数引数が `agent` なら関数内では `agent.xxx` を使用
  - `ag.xxx` のような省略変数を混在させない
- **Canvas描画ループ全体を try-catch で保護**すること
  - 1箇所のエラーが全キャラの描画停止を引き起こすため
- **アイソメトリック描画の深度ソート**を必ず実装
  - Y座標が大きい（手前の）オブジェクトを後に描画する
- 全UI・全メッセージ・全ログは**日本語**で表記すること
- 英語の変数名・関数名はOK、ユーザーに見えるテキストは全て日本語
